# プロジェクトレポート: RSS News Portal

## プロジェクト名称

**RSS News Portal**

## プロジェクト概要

国内外の主要なRSSフィード（Yahoo!ニュース、NHK、BBC、技術系ブログ等）を定期的に収集し、カテゴリー別に整理された静的なHTMLダッシュボードを自動生成するシステムです。GitHub Actionsによる自動実行とGitHub Pagesによる公開を組み合わせることで、サーバーレスかつ低コストな個人専用ニュースポータルの構築を目的としています。

## 主要な技術スタック

* **プログラミング言語**: Python 3
* **テンプレートエンジン**: Jinja2
* **RSSパース**: feedparser
* **CI/CD・デプロイ**: GitHub Actions
* **フロントエンド**: HTML5, Bootstrap 5 (CSSフレームワーク)

## ディレクトリ構造

```text
RSS/
├── .github/
│   └── workflows/
│       └── update_rss.yml    # 定期実行ワークフロー
├── docs/                      # 生成された公開用HTML（GitHub Pages）
│   ├── index.html             # 総合ポータル画面
│   ├── tech.html              # 技術ニュース
│   ├── business.html          # 経済ニュース
│   └── ...                    # その他カテゴリー別HTML
├── build.py                   # ニュース収集・ビルドスクリプト
├── template.html              # 出力用HTMLテンプレート
└── requirements.txt           # 依存ライブラリ定義

```

## ファイル一覧と解説

### コア・ロジック

* **`build.py`**
* **役割**: システムのバックエンド・オーケストレーター。
* **責務**: `FEEDS` 定数に定義されたRSSソースから情報を取得し、Jinja2を用いて `template.html` をレンダリングします。全ニュースの概要をまとめた「ポータル画面」と、各ジャンルに特化した「カテゴリー画面」をそれぞれ静的ファイルとして出力します。



### プレゼンテーションレイヤー

* **`template.html`**
* **役割**: フロントエンドのレイアウト定義。
* **責務**: Bootstrap 5を使用したレスポンシブデザインを提供します。サイドバーによるナビゲーション、記事のカード表示、時刻表示等のUIコンポーネントを保持し、Pythonから注入されたニュースデータを可視化します。



### 自動化・インフラ

* **`.github/workflows/update_rss.yml`**
* **役割**: 自動更新パイプライン。
* **責務**: GitHub Actions上で6時間おきに起動し、`build.py` を実行して最新のニュースを収集・ビルドします。生成されたHTMLファイルをGitHubリポジトリへ自動的にコミット・プッシュし、GitHub Pagesの表示内容を更新します。



## エントリーポイント

プログラムの実行は、GitHub Actionsによってトリガーされる `build.py` が起点となります。

**動作の流れ**:

1. **スケジュール起動**: 定期的なスケジュール（Cron）または手動Pushを検知してワークフローが開始されます。
2. **環境構築**: 仮想環境上でPythonがセットアップされ、`feedparser` や `Jinja2` などの依存パッケージがインストールされます。
3. **収集・変換**: `build.py` が実行され、全RSSソースを並列的に巡回して最新の記事タイトル、リンク、概要を抽出します。
4. **HTMLビルド**: 取得したデータが各カテゴリー（Tech, Business, Yahoo等）ごとに整理され、`docs/` ディレクトリ内に個別のHTMLファイルとして書き出されます。
5. **デプロイ**: 成果物がリポジトリに反映され、GitHub Pagesを通じてインターネット上に公開されます。
